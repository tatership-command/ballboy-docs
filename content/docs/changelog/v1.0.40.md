# v1.0.40 — Live Scores in the Weekly Announcement & Cleaner Stream Posts

_Released 2026-07-20_

## What's New

**📊 The weekly announcement now updates as games finish**
Your week announcement used to be a snapshot — posted once, frozen with every game showing "vs" even after results came in. Now it updates itself. As each game goes final, the Games list re-renders in place with the score. It's the same message, edited quietly: nobody gets re-pinged, and the 🔔/🔕 buttons stay put. Works the same whether a result comes from `/season result`, the Game Played button, or an EA export.

**🔴 Go Live and `/stream` post one link, not all of them**
If you had both Twitch and YouTube linked, going live posted both. Now you get exactly one — the default you picked in the claim window, or your first linked account if you never set one. Pick your default any time by re-running the claim/connect flow.

**📌 `/stream` also posts in your game thread**
When you're streaming a game that's on this week's schedule, `/stream` now posts the announcement in that game's thread too and pins it there, on top of wherever it normally lands.

## Fixes

**🎨 Go Live now uses your league's color**
The Go Live announcement always came out default orange, ignoring the color set with `/admin board_color`. It now matches the rest of your league's embeds.

**📌 Forum leagues get a pinned game post again**
If your `game_thread` channel is a forum, the starter post wasn't being pinned. It is now — matching how text channels have always worked.

**🔐 Ball Boy checks for the new Pin Messages permission**
Discord split "Pin Messages" out of "Manage Messages" into its own permission earlier this year, and existing bots were never granted it — which is why pinning had started failing. `/admin roles setup` now checks for it and tells you if it's missing.

> **Action needed if pins aren't working:** grant Ball Boy **Pin Messages** under Server Settings → Integrations → Ball Boy. Run `/admin roles setup` to confirm.
